## Architecture Overview

### Layer Structure
```
┌─────────────────────────────────────────┐
│     MCP Tools (@mcp.tool())             │ ← User-facing interface
├─────────────────────────────────────────┤
│     Tool Logic & Orchestration          │ ← Coordination layer
├─────────────────────────────────────────┤
│     SumoLogicClient (API Methods)       │ ← API abstraction
├─────────────────────────────────────────┤
│     Validation, Config, Rate Limiting   │ ← Cross-cutting concerns
├─────────────────────────────────────────┤
│     Sumo Logic REST APIs                │ ← External dependency
└─────────────────────────────────────────┘
```

### Design Principles

1. **Single Responsibility**: Each module has one clear purpose
2. **DRY (Don't Repeat Yourself)**: Reuse code through `SumoLogicClient` and helper functions
3. **Fail Fast**: Validate inputs early, before making API calls
4. **Explicit Dependencies**: All configuration comes from `.env`, passed through `config.py`
5. **Defensive Programming**: Always handle errors, never expose sensitive data
6. **Read-Only Operations**: No write/update/delete operations, only GET requests (except async job creation)

## API Client Pattern

### Purpose
Centralize all Sumo Logic API interactions in the `SumoLogicClient` class to ensure consistency, reusability, and maintainability.

### Pattern Structure

```python
class SumoLogicClient:
    """Client for interacting with Sumo Logic APIs."""

    def __init__(self, instance_config: SumoInstanceConfig, instance_name: str = "default"):
        """Initialize client with instance configuration."""
        self.instance_name = instance_name
        self.access_id = instance_config.access_id
        self.access_key = instance_config.access_key
        self.endpoint = instance_config.endpoint
        self.session = httpx.AsyncClient(
            auth=(instance_config.access_id, instance_config.access_key),
            headers={"Content-Type": "application/json"},
            timeout=30.0
        )

    async def _request(
        self,
        method: str,
        path: str,
        api_version: str = "v1",
        **kwargs
    ) -> Dict[str, Any]:
        """
        Make an authenticated request to the Sumo Logic API.

        - Builds full URL with API version
        - Handles authentication automatically
        - Logs requests for audit
        - Converts HTTP errors to custom exceptions
        - Returns JSON response as dict
        """
        # Implementation handles all error cases
        pass

    async def get_specific_resource(self, param1: str, param2: int = 100) -> Dict[str, Any]:
        """
        Get specific resource from Sumo Logic.

        Args:
            param1: Description of parameter
            param2: Description with default value

        Returns:
            API response as dictionary

        API Endpoint: GET /api/v1/resource
        Documentation: https://api.sumologic.com/docs/#operation/getResource
        """
        params = {"param1": param1, "limit": param2}
        return await self._request("GET", "/resource", api_version="v1", params=params)
```

### Rules

1. **All API calls go through `SumoLogicClient`**
   - NEVER make direct `httpx` calls in tool functions
   - Add a new method to `SumoLogicClient` if endpoint doesn't exist

2. **Method naming convention**
   ```python
   # GET operations
   async def get_dashboards() -> Dict[str, Any]:
   async def list_collectors() -> Dict[str, Any]:
   async def search_monitors() -> Dict[str, Any]:

   # POST operations (async jobs only)
   async def begin_content_export() -> Dict[str, Any]:
   ```

3. **API Version Handling**
   ```python
   # Sumo Logic API versions by endpoint:
   # - Search Jobs: v1
   # - Collectors & Sources: v1
   # - Users: v1
   # - Content: v2
   # - Dashboards: v2
   # - Roles: v2
   # - Metrics: v1 (but use /metrics/queries path)

   return await self._request("GET", "/content", api_version="v2")
   ```

4. **Always document the API endpoint**
   ```python
   async def get_account_status(self) -> Dict[str, Any]:
       """
       Get account status and subscription information.

       API Endpoint: GET /api/v1/account/status
       Documentation: https://api.sumologic.com/docs/#operation/getStatus
       """
   ```

### Example: Adding New API Method

```python
# ❌ WRONG: Direct API call in tool
@mcp.tool()
async def get_wrong_way():
    async with httpx.AsyncClient() as client:
        response = await client.get(url, auth=(id, key))
        return response.json()

# ✅ CORRECT: API method + tool
class SumoLogicClient:
    async def get_partitions(self, limit: int = 100, offset: int = 0) -> Dict[str, Any]:
        """
        Get list of partitions.

        API Endpoint: GET /api/v1/partitions
        """
        params = {"limit": limit, "offset": offset}
        return await self._request("GET", "/partitions", api_version="v1", params=params)

@mcp.tool()
async def get_sumo_partitions(limit: int = 100, instance: str = 'default') -> str:
    """Get list of partitions."""
    try:
        client = await get_sumo_client(instance)
        result = await client.get_partitions(limit=limit)
        return json.dumps(result, indent=2)
    except Exception as e:
        return handle_tool_error(e, "get_sumo_partitions")
```

## MCP Tool Pattern

### Purpose
Standardize the structure of all MCP tools for consistency, error handling, and maintainability.

### Standard Tool Template

```python
@mcp.tool()
async def tool_name(
    # Required parameters (no defaults)
    param1: str = Field(description="Clear description of what this is"),

    # Optional parameters (with defaults)
    param2: int = Field(default=100, description="Description with default value"),
    param3: Optional[str] = Field(default=None, description="Optional parameter"),

    # Always last: instance parameter
    instance: str = Field(default='default', description="Sumo Logic instance name")
) -> str:  # Always return str (JSON formatted)
    """
    Brief one-line description of what the tool does.

    Longer description explaining:
    - Purpose and use cases
    - What data it returns
    - Any important notes or limitations

    Example:
    - Show typical usage or output format
    """
    try:
        # 1. INITIALIZE: Config and rate limiting
        _ensure_config_initialized()
        config = get_config()
        limiter = get_rate_limiter(config.server_config.rate_limit_per_minute)
        await limiter.acquire("tool_name")

        # 2. VALIDATE: All inputs
        instance = validate_instance_name(instance)
        param1 = validate_query_input(param1)  # Use appropriate validator
        param2 = validate_pagination(limit=param2, offset=0)  # If pagination

        # 3. EXECUTE: Get client and call API
        client = await get_sumo_client(instance)
        result = await client.api_method(param1, param2)

        # 4. TRANSFORM: Process result if needed
        # (Optional) Add computed fields, filter data, format output

        # 5. RETURN: Always JSON string, pretty printed
        return json.dumps(result, indent=2)

    except Exception as e:
        # ALWAYS use handle_tool_error for consistent error handling
        return handle_tool_error(e, "tool_name")
```

### Tool Organization in File

Tools are organized by category in `sumologic_mcp_server.py`:

```python
# ============================================================================
# SEARCH & QUERY TOOLS
# ============================================================================

@mcp.tool()
async def search_sumo_logs(...):
    pass

@mcp.tool()
async def create_sumo_search_job(...):
    pass

# ============================================================================
# CONTENT LIBRARY TOOLS
# ============================================================================

@mcp.tool()
async def get_personal_folder(...):
    pass

# ... etc
```

### Parameter Ordering Rules

1. Required parameters first (no defaults)
2. Optional parameters second (with defaults)
3. `instance` parameter always last
4. Use descriptive parameter names (no single letters except for common abbreviations)

```python
# ✅ GOOD
async def search_sumo_logs(
    query: str = Field(description="Search query"),
    hours_back: int = Field(default=1, description="Hours to search back"),
    from_time: Optional[str] = None,
    to_time: Optional[str] = None,
    instance: str = Field(default='default', description="Instance name")
) -> str:

# ❌ BAD
async def search_sumo_logs(
    instance: str = 'default',  # Should be last
    q: str,  # Bad name, should be 'query'
    h: int = 1  # Bad name, should be 'hours_back'
) -> Dict:  # Should return str (JSON)
```

## Error Handling Pattern

### Exception Hierarchy

```
SumoMCPError (base)
├── ValidationError      # Input validation failures
├── AuthenticationError  # 401, 403 errors
├── APIError            # General API errors (4xx, 5xx)
└── TimeoutError        # Request timeouts
```

### Error Handling in Tools

```python
@mcp.tool()
async def example_tool(param: str, instance: str = 'default') -> str:
    try:
        # Tool logic here
        pass

    except ValidationError as e:
        # Validation errors are already well formatted
        return handle_tool_error(e, "example_tool")

    except AuthenticationError as e:
        # Auth errors are already sanitized
        return handle_tool_error(e, "example_tool")

    except APIError as e:
        # API errors have status codes and details
        return handle_tool_error(e, "example_tool")

    except Exception as e:
        # Catch-all for unexpected errors
        return handle_tool_error(e, "example_tool")
```

### Error Response Format

The `handle_tool_error()` function returns consistent JSON error responses:

```json
{
  "error": "ValidationError",
  "message": "Query exceeds maximum length of 10000 characters",
  "tool": "search_sumo_logs",
  "details": "query_length=15000"
}
```

### Error Handling in SumoLogicClient

```python
async def _request(self, method: str, path: str, **kwargs) -> Dict[str, Any]:
    """Make API request with comprehensive error handling."""
    try:
        response = await self.session.request(method, url, **kwargs)
        response.raise_for_status()
        return response.json()

    except httpx.HTTPStatusError as e:
        # Convert HTTP errors to custom exceptions
        if e.response.status_code == 401:
            raise AuthenticationError("Authentication failed...")
        elif e.response.status_code == 403:
            raise AuthenticationError("Access denied...")
        elif e.response.status_code == 429:
            raise APIError("Rate limit exceeded...", status_code=429)
        else:
            raise APIError(f"Request failed with status {status_code}",
                          status_code=status_code)

    except httpx.TimeoutException:
        raise TimeoutError("Request to Sumo Logic API timed out")

    except Exception as e:
        raise APIError("Unexpected error communicating with API")
```

### Rules

1. **Never expose sensitive data in errors**
   - Don't include API keys, access IDs, or passwords
   - Sanitize URLs to remove credentials

2. **Always use `handle_tool_error()`**
   - Provides consistent error format
   - Logs errors appropriately
   - Sanitizes error messages

3. **Log errors for debugging**
   ```python
   logger.error(f"Request failed for {self.instance_name}: {str(e)}")
   ```

4. **Provide actionable error messages**
   ```python
   # ❌ BAD
   raise ValidationError("Invalid input")

   # ✅ GOOD
   raise ValidationError(
       f"Query exceeds maximum length of 10000 characters. "
       f"Your query is {len(query)} characters."
   )
   ```

## Validation Pattern

### Purpose
Validate all inputs before making API calls to fail fast and provide clear error messages.

### Validation Functions

```python
# From validation.py

def validate_instance_name(instance: str) -> str:
    """Validate instance name format."""
    if not re.match(r'^[a-zA-Z0-9_-]+$', instance):
        raise ValidationError("Instance name must be alphanumeric with _ or -")
    return instance

def validate_query_input(query: str) -> str:
    """Validate query string."""
    if not query or not query.strip():
        raise ValidationError("Query cannot be empty")
    if len(query) > 10000:
        raise ValidationError(f"Query exceeds maximum length of 10000 characters")
    if '\x00' in query:
        raise ValidationError("Query contains null bytes")
    return query.strip()

def validate_time_range(hours: int) -> int:
    """Validate time range."""
    if hours < 0:
        raise ValidationError("hours_back must be positive")
    if hours > 8760:  # 365 days
        raise ValidationError("hours_back cannot exceed 8760 hours (1 year)")
    if hours > 720:  # 30 days
        logger.warning(f"Large time range: {hours} hours")
    return hours

def validate_pagination(limit: int, offset: int) -> tuple[int, int]:
    """Validate pagination parameters."""
    if limit < 1 or limit > 1000:
        raise ValidationError("limit must be between 1 and 1000")
    if offset < 0 or offset > 100000:
        raise ValidationError("offset must be between 0 and 100000")
    return limit, offset
```

### Validation Classes (Pydantic)

For complex validation:

```python
from pydantic import BaseModel, Field, field_validator

class CollectorValidation(BaseModel):
    """Validation model for collector operations."""

    collector_id: int = Field(..., gt=0)

    @field_validator('collector_id')
    @classmethod
    def validate_collector_id(cls, v: int) -> int:
        if v <= 0:
            raise ValueError("collector_id must be positive")
        return v
```

### Usage in Tools

```python
@mcp.tool()
async def search_sumo_logs(
    query: str = Field(description="Search query"),
    hours_back: int = Field(default=1, description="Hours back"),
    instance: str = Field(default='default', description="Instance")
) -> str:
    try:
        # Validate ALL inputs before API calls
        instance = validate_instance_name(instance)
        query = validate_query_input(query)
        hours_back = validate_time_range(hours_back)

        # Now safe to proceed with API calls
        client = await get_sumo_client(instance)
        # ...
```

### Rules

1. **Validate before API calls** - Fail fast to save API quota
2. **Provide clear error messages** - Tell user what's wrong and how to fix it
3. **Log warnings for edge cases** - e.g., large time ranges, high limits
4. **Use type hints** - Help catch errors at development time

## Configuration Pattern

### Purpose
Centralize all configuration in `.env` file and provide type-safe access through `config.py`.

### Configuration Structure

```python
# config.py

class SumoInstanceConfig(BaseModel):
    """Configuration for a single Sumo Logic instance."""

    access_id: str = Field(..., min_length=1)
    access_key: str = Field(..., min_length=1)
    endpoint: str = Field(...)
    subdomain: Optional[str] = Field(default=None)

    @field_validator('endpoint')
    @classmethod
    def validate_endpoint(cls, v: str) -> str:
        if not v.startswith(('https://', 'http://')):
            raise ValueError("Endpoint must start with https:// or http://")
        return v.rstrip('/')

class Config:
    """Application configuration manager."""

    def __init__(self):
        self.instances: Dict[str, SumoInstanceConfig] = {}
        self.server_config = self._load_server_config()
        self._load_instances()

    def get_instance(self, instance_name: str = 'default') -> SumoInstanceConfig:
        """Get configuration for specific instance."""
        if instance_name not in self.instances:
            raise ValueError(f"Instance '{instance_name}' not found")
        return self.instances[instance_name]
```

## Async Operations Pattern

### Purpose
Handle long-running Sumo Logic operations (exports, large searches) with proper async/await patterns and polling.

### Async Job Pattern

Many Sumo Logic operations are asynchronous:
1. Create job → Get job ID
2. Poll job status → Wait for completion
3. Retrieve result → Get final data

### Implementation Pattern

```python
# async_export_helper.py

async def poll_export_job(
    client: SumoLogicClient,
    content_id: str,
    is_admin_mode: bool = False,
    max_wait_seconds: int = 300,
    poll_interval: int = 2
) -> Dict[str, Any]:
    """
    Poll an async export job until completion.

    Args:
        client: Authenticated Sumo Logic client
        content_id: Content ID to export
        is_admin_mode: Use admin mode for export
        max_wait_seconds: Maximum time to wait
        poll_interval: Seconds between polls

    Returns:
        Export result data

    Raises:
        TimeoutError: If job doesn't complete in time
        APIError: If job fails
    """
    # 1. Start the job
    job_response = await client.begin_content_export(content_id, is_admin_mode)
    job_id = job_response['id']

    # 2. Poll for completion
    max_attempts = max_wait_seconds // poll_interval
    for attempt in range(max_attempts):
        status_response = await client.get_content_export_status(content_id, job_id)
        status = status_response['status']

        if status == 'Success':
            # 3. Get result
            result = await client.get_content_export_result(content_id, job_id)
            return result
        elif status == 'Failed':
            raise APIError(f"Export job failed: {status_response.get('error')}")

        # Wait before next poll
        await asyncio.sleep(poll_interval)

    raise TimeoutError(f"Export job did not complete within {max_wait_seconds} seconds")
```

### Usage in Tools

```python
@mcp.tool()
async def export_content(
    content_id: str = Field(description="Content ID"),
    max_wait_seconds: int = 300,
    instance: str = Field(default='default', description="Instance")
) -> str:
    """Export content with async job handling."""
    try:
        from .async_export_helper import poll_export_job

        client = await get_sumo_client(instance)
        result = await poll_export_job(
            client,
            content_id,
            is_admin_mode=False,
            max_wait_seconds=max_wait_seconds
        )

        return json.dumps(result, indent=2)
    except Exception as e:
        return handle_tool_error(e, "export_content")
```

### Rules

1. **Use helper functions** - Don't repeat polling logic
2. **Configurable timeouts** - Let user control max wait time
3. **Reasonable poll intervals** - 2-5 seconds is typical
4. **Clear progress indication** - Log polling attempts
5. **Handle failures gracefully** - Check for 'Failed' status

## URL Generation Pattern

### Purpose
Generate correct web UI URLs for different Sumo Logic regions, subdomains, and content types.

### URL Structure

```
API Endpoint → Web UI URL

https://api.au.sumologic.com → https://service.au.sumologic.com
https://api.au.sumologic.com (with subdomain) → https://mycompany.au.sumologic.com
```

### URL Builder Pattern

```python
# url_builder.py

def get_ui_base_url(api_endpoint: str, subdomain: Optional[str] = None) -> str:
    """
    Convert API endpoint to web UI base URL.

    Maps API endpoints to UI endpoints per region.
    Supports custom subdomains.
    """
    clean_endpoint = api_endpoint.rstrip('/').replace('/api', '')

    if clean_endpoint in API_TO_UI_MAPPING:
        ui_host = API_TO_UI_MAPPING[clean_endpoint]
        if subdomain:
            ui_host = subdomain + ui_host[7:]  # Replace 'service'
        return f'https://{ui_host}'

    # Fallback for unknown endpoints
    if 'api.' in clean_endpoint:
        if subdomain:
            return clean_endpoint.replace('api.', f'{subdomain}.')
        return clean_endpoint.replace('api.', 'service.')

    return clean_endpoint

def build_library_url(api_endpoint: str, decimal_id: str, subdomain: Optional[str] = None) -> str:
    """Build library content URL."""
    base = get_ui_base_url(api_endpoint, subdomain)
    return f'{base}/library/{decimal_id}'

def build_dashboard_url(api_endpoint: str, dashboard_id: str, subdomain: Optional[str] = None) -> str:
    """Build dashboard URL."""
    base = get_ui_base_url(api_endpoint, subdomain)
    return f'{base}/dashboard/{dashboard_id}'

def build_search_url(
    api_endpoint: str,
    query: str,
    start_time: str = '-1h',
    end_time: str = '-1s',
    subdomain: Optional[str] = None
) -> str:
    """Build search URL with query and time range."""
    base = get_ui_base_url(api_endpoint, subdomain)
    params = urlencode({'query': query, 'startTime': start_time, 'endTime': end_time})
    return f'{base}/log-search/create?{params}'
```

### Usage Pattern

```python
@mcp.tool()
async def get_content_web_url(
    content_id: str,
    content_type: Optional[str] = None,
    instance: str = 'default'
) -> str:
    """Generate web UI URL for content."""
    from .url_builder import build_library_url, build_dashboard_url
    from .content_id_utils import hex_to_decimal, normalize_to_hex

    # Get configuration
    instance_config = config.get_instance(instance)
    subdomain = instance_config.subdomain

    # Convert ID format
    hex_id = normalize_to_hex(content_id)
    decimal_id = hex_to_decimal(hex_id)

    # Generate appropriate URL based on content type
    if content_type == 'Dashboard':
        # Dashboards use unique ID, not content ID
        dashboard_id = await get_dashboard_unique_id(hex_id)
        url = build_dashboard_url(instance_config.endpoint, dashboard_id, subdomain)
    else:
        # Library content uses decimal content ID
        url = build_library_url(instance_config.endpoint, decimal_id, subdomain)

    return json.dumps({"url": url, "hex_id": hex_id, "decimal_id": decimal_id})
```

### Rules

1. **Use url_builder.py** - Don't construct URLs manually in tools
2. **Support all regions** - Test with different endpoints
3. **Handle subdomains** - Check configuration for custom subdomain
4. **URL encode parameters** - Use `urllib.parse.urlencode()`
5. **Document URL format** - Show examples in docstrings

## Testing Patterns

### Test Structure

```python
"""Tests for module_name functionality."""

import pytest
from src.sumologic_mcp_server.module_name import function_name


class TestFunctionName:
    """Tests for function_name function."""

    def test_basic_functionality(self):
        """Test basic use case."""
        result = function_name("input")
        assert result == "expected"

    def test_with_optional_params(self):
        """Test with optional parameters."""
        result = function_name("input", param2="custom")
        assert "custom" in result

    def test_edge_cases(self):
        """Test edge cases and boundary conditions."""
        assert function_name("") == ""
        assert function_name("x" * 10000) is not None

    def test_error_handling(self):
        """Test error conditions."""
        with pytest.raises(ValueError):
            function_name(None)


class TestClassOrModule:
    """Tests for ClassOrModule."""

    @pytest.fixture
    def sample_data(self):
        """Fixture for test data."""
        return {"key": "value"}

    def test_initialization(self, sample_data):
        """Test initialization with fixtures."""
        obj = ClassOrModule(sample_data)
        assert obj.key == "value"
```

### Integration Test Pattern

```python
"""Integration tests for tool functionality."""

import pytest
import asyncio

@pytest.mark.asyncio

        assert "error" not in result

    async def test_tool_with_custom_instance(self):
        """Test tool with custom instance."""
        result = await tool_name("input", instance="demo")
        assert "demo" in result or "error" not in result

    async def test_tool_error_handling(self):
        """Test tool error handling."""
        result = await tool_name("")  # Invalid input
        assert "error" in result
```

### Test Coverage Goals

- **Unit tests**: 80%+ coverage for utility functions
- **Integration tests**: Critical path coverage for major tools
- **Error handling**: Test all error conditions
- **Edge cases**: Test boundary conditions

### Rules

1. **Test what matters** - Focus on business logic, not trivial code
2. **Use descriptive names** - Test name should describe what it tests
3. **One assertion per test** - Or at least one logical concept
4. **Use fixtures** - For setup/teardown and shared data
5. **Mock external APIs** - Don't make real API calls in unit tests
6. **Test error paths** - Ensure errors are handled gracefully
